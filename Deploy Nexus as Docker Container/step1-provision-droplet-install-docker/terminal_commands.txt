# 📦 SUMMARY  
# These commands connect to a remote server (Droplet) via SSH, update the package list, and install Docker using Snap package manager.  
# Language/Platform: Bash (Linux) and SSH  
# What you'll learn: Remote server access, package management, Docker installation via Snap  
# Outcome: Docker installed and ready to use on your remote Ubuntu/Debian-based Droplet  
# Why this matters: Setting up Docker on a remote server is essential for deploying containerized applications in production or testing environments  

# ------------------------------  
# 🧩 SECTION TITLE: Connect to Remote Server via SSH  
# ------------------------------  
ssh root@<your-droplet-ip>  
# ✅ What it does: Opens a secure shell session to your remote server as the root user at the specified IP address  
# 🧠 Why it matters: Provides remote command-line access to manage your server  
# 🔄 Equivalent PowerShell command:  
# ssh root@<your-droplet-ip>  
# Explanation: PowerShell supports ssh the same way if OpenSSH client is installed  

# ------------------------------  
# 🧩 SECTION TITLE: Update Package List  
# ------------------------------  
sudo apt update  
# ✅ What it does: Refreshes the local package index to the latest versions available from configured repositories  
# 🧠 Why it matters: Ensures you install the latest available versions of software packages, including dependencies  
# 🔄 Equivalent PowerShell command: N/A (runs on remote Linux server)  

# ------------------------------  
# 🧩 SECTION TITLE: Install Docker via Snap  
# ------------------------------  
snap install docker  
# ✅ What it does: Installs Docker using Snap, a universal Linux package manager, which handles dependencies and updates automatically  
# 🧠 Why it matters: Provides a quick, easy, and sandboxed Docker installation on supported Linux distros  
# 🔄 Equivalent PowerShell command: N/A (runs on remote Linux server)  

# ------------------------------  
# ✅ NOTES / GOTCHAS / TIPS  
# ------------------------------  
# - Replace `<your-droplet-ip>` with your actual server IP address or hostname.  
# - If you don’t have SSH key setup, you will be prompted for the root password.  
# - Installing Docker via Snap is straightforward but sometimes installing via the official Docker repos can provide newer versions.  
# - After installation, verify Docker is running with `sudo docker version` or `sudo docker info`.  
# - Snap installs Docker with root permissions; you might need to run Docker commands with `sudo` unless you configure Docker group permissions.

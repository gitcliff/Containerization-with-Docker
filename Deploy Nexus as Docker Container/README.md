# Deploy Nexus as Docker Container

# 📖 Summary

Worked on **Deploy Nexus as Docker Container**, focusing on **containerization, cloud provisioning, and persistent data management**.
Tasks included **provisioning a cloud server, installing Docker, deploying Nexus 3 as a container, and verifying persistent volumes** to solve **the challenge of hosting a private Docker registry with data persistence in the cloud**.
Set up **Docker and Nexus on a remote droplet** to automate and optimize the private registry deployment, aiming to **enable secure, scalable, and manageable Docker image hosting**.

---

# 🛠️ Tools Used

* **Droplet (Cloud VM)**: ✨ *Provisioned a remote Linux server for container hosting*
* **Docker (via Snap)**: ⚙️ *Installed and managed containers on the remote server*
* **Sonatype Nexus 3**: 🚀 *Deployed as a private Docker registry inside a container with persistent storage*

---

# 🎯 Skills Gained

* **Remote Server Management**: 💡 *Used SSH to securely connect and configure cloud servers*
* **Docker Container Deployment**: 🔍 *Deployed and managed Nexus 3 as a containerized application with volume mounts*
* **Persistent Storage Handling**: 🚀 *Created and inspected Docker volumes ensuring data durability across container lifecycles*

---

# ⚠️ Challenges Faced

* **Ensuring Port Accessibility**: 🛠️ *Configured firewall and verified open ports to access Nexus UI externally*
* **Docker Snap Installation Permissions**: 🔄 *Managed user permissions to run Docker commands smoothly after Snap install*

---

# 💡 Why It Matters

This lab strengthens expertise in **DevOps and cloud infrastructure management**, demonstrating how tools like **Docker and Nexus** improve software delivery through containerization and centralized artifact storage.
Mastering these skills supports efficient, reliable CI/CD practices essential in modern DevOps and cloud roles. 🚀🌐
